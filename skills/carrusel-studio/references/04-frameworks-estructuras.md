# KB_04 — FRAMEWORKS Y ESTRUCTURAS
## Library de tipos de carrusel y narrativas — adaptado a marcas variadas

---

## 1. CÓMO USAR ESTE DOCUMENTO

Cada vez que el usuario trae un brief, sigue 3 pasos:

1. **Identifica el TIPO de carrusel** (sección 2)
2. **Elige el FRAMEWORK narrativo** (sección 3)  
3. **Aplica la ESTRUCTURA de slides** (sección 4)

Las estructuras son **plantillas, no jaulas**. Adapta el número de slides (8-12) al contenido. Lo importante es respetar la lógica narrativa.

**SIEMPRE explica al usuario por qué eliges X tipo y X framework.** El usuario está aprendiendo.

---

## 2. TIPOS DE CARRUSEL (6 categorías)

### 2.1 EDUCATIVO / MÉTODO
Enseñas un proceso, framework, sistema o conjunto de reglas.
- **Cuándo:** Tienes una metodología validada y quieres compartirla
- **Objetivo principal:** Saves + autoridad
- **Ejemplos:** "5 pasos para X", "Las 4 reglas que aplico para Y"
- **Slides típicos:** 9-11

### 2.2 CASE STUDY
Cuentas un caso real de un proyecto o cliente.
- **Cuándo:** Tienes resultados concretos y permiso para mostrarlos
- **Objetivo:** Autoridad + credibilidad + leads
- **Ejemplos:** "Cómo rebrandeamos a [X]", "El proyecto que casi fallamos"
- **Slides típicos:** 8-10

### 2.3 LISTA / LISTICLE
Listado numerado de items (herramientas, errores, tips, recursos).
- **Cuándo:** Valor en cantidad/variedad más que profundidad
- **Objetivo:** Saves + shares
- **Ejemplos:** "10 herramientas X", "5 errores comunes en Y"
- **Slides típicos:** N+3

### 2.4 STORYTELLING / NARRATIVA
Cuentas una historia con principio, conflicto y resolución.
- **Cuándo:** Tienes una experiencia con tensión narrativa real
- **Objetivo:** Engagement + conexión
- **Ejemplos:** "Cómo perdí mi mejor cliente", "El día que decidí X"
- **Slides típicos:** 8-10

### 2.5 CONTRARIAN / MYTHBUSTING
Defiendes postura impopular o derribas creencia común.
- **Cuándo:** Tienes datos/experiencia para sostener opinión punzante
- **Objetivo:** Shares (alta viralidad) + autoridad
- **Ejemplos:** "Por qué X está sobrevalorado", "No necesitas Y"
- **Slides típicos:** 7-9

### 2.6 ANUNCIO / LANZAMIENTO
Anuncias algo nuevo (servicio, producto, evento).
- **Cuándo:** Tienes algo concreto que vender
- **Objetivo:** Conversión / clicks / DMs
- **Ejemplos:** "Abrí X lugares para nuevos clientes", "Nuevo curso: [Y]"
- **Slides típicos:** 6-8

## 3. FRAMEWORKS NARRATIVOS

Cada framework es una secuencia emocional. Elegir framework = elegir cómo se siente el carrusel.

### Framework 1 — AIDA
**Atención → Interés → Deseo → Acción**

| Fase | Función | Slides |
|---|---|---|
| Atención | Hook que detiene scroll | S1 |
| Interés | Información que genera curiosidad | S2-S3 |
| Deseo | Mostrar el beneficio o resultado | S4-S7 |
| Acción | CTA específico | S8-S10 |

**Best for:** Educativo, anuncio, lista

### Framework 2 — PAS
**Problema → Agitación → Solución**

| Fase | Función | Slides |
|---|---|---|
| Problema | Identifica el dolor | S1-S2 |
| Agitación | Profundiza el dolor | S3-S4 |
| Solución | Tu propuesta | S5-S9 |
| Cierre | CTA | S10 |

**Best for:** Educativo, contrarian, anuncio

### Framework 3 — BAB
**Before → After → Bridge**

| Fase | Función | Slides |
|---|---|---|
| Before | Cómo era antes | S1-S3 |
| After | Cómo es ahora | S4-S6 |
| Bridge | El cómo (pasos) | S7-S9 |
| CTA | Acción | S10 |

**Best for:** Case study, transformación

### Framework 4 — Hero's Journey condensado
**Llamada → Conflicto → Aliado → Transformación**

| Fase | Función | Slides |
|---|---|---|
| Status quo | Cómo estaba todo | S1-S2 |
| Inciting incident | El cambio | S3 |
| Crisis | El conflicto | S4-S5 |
| Aliado/insight | Lo que cambió | S6-S7 |
| Transformación | El después | S8-S9 |
| CTA | Lección/acción | S10 |

**Best for:** Storytelling, case study con narrativa fuerte

### Framework 5 — Listicle puro
**Intro → Item 1 → Item N → Resumen → CTA**

**Best for:** Lista, recursos, herramientas

### Framework 6 — Contrarian / Mythbusting
**Premisa común → Por qué es falsa → La verdad → Implicación**

**Best for:** Contrarian, takes punzantes

## 5. DECISION TREE — ¿QUÉ ESTRUCTURA USO?

Pregúntate en este orden:

1. **¿El carrusel anuncia algo?** → ESTRUCTURA G
2. **¿Es una historia con conflicto?** → ESTRUCTURA E
3. **¿Defiende postura impopular?** → ESTRUCTURA F
4. **¿Es un caso real de proyecto?** → ESTRUCTURA C
5. **¿Es una lista de N items independientes?** → ESTRUCTURA D
6. **¿Enseña un proceso paso a paso?** → ESTRUCTURA B
7. **¿Enseña reglas/principios independientes?** → ESTRUCTURA A

## 6. NÚMERO DE SLIDES — RANGOS

| Tipo | Ideal | Min | Max |
|---|---|---|---|
| Educativo | 9-11 | 7 | 12 |
| Case study | 8-10 | 6 | 11 |
| Lista | N+3 | N+2 | N+4 |
| Storytelling | 8-10 | 7 | 11 |
| Contrarian | 7-9 | 6 | 10 |
| Anuncio | 6-8 | 5 | 9 |

Si pasas de 12, divide en 2 carruseles serie.

---

## 7. ELEMENTOS NARRATIVOS UNIVERSALES

Todo carrusel necesita estos 6 elementos:

| Elemento | Función | Slide típico |
|---|---|---|
| Hook | Detener scroll | S1 |
| Setup | Contextualizar quién/qué/por qué | S2-S3 |
| Tesis | Propuesta central | S3 o S4 |
| Evidencia | Datos, ejemplos, items | S4-S(N-2) |
| Insight | "Aha moment" o lección | S(N-2) o S(N-1) |
| CTA | Acción concreta | S(N) |

Si tu estructura no tiene alguno, el carrusel se siente incompleto.
